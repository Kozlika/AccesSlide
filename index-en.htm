AccesSlide
Finally an accessible slideshow

by Access42

Structure

Slides are to be put in the main element with section elements associated with a slide class.

The first and the last slide have an additional class : couv and end class.

section elements can be imbricated in section elements.

Short example of structure

<section class="slide">
 <h2>Title of the slide</h2>
 [content]
</section>

<section class="slide" aria-label="title of the slide">
 [content]
</section>

Navigation

The navigation bar has (in order of appearance):

  1. a previous button
  2. a selection list to reach a slide by its index
  3. a next button
  4. a table of contents to reach a slide through its title
  5. the index (number of the current slide and total of slides)
  6. a configuration button

Keyboard navigation

  • SPACE or RIGHT ARROW or Click Next slide
  • SHIFT + SPACE or LEFT ARROW Previous slide
  • Start First slide
  • End Last slide
  • ALT + 0 (zero) Table of contents

To go to the next slide with Jaws : ignore the next keystroke (using INSERT + 3) then press SPACE to make the slideshow scroll.

With NVDA, ignoring the next key is not necessary, the SPACE keystroke works.

Effects

The available effects can be set/defined in the configuration panel.

To create your own effects:

  1. Create a class, for example .my-effect
  2. In the AccesSlide.js file, create an entry in the object config (see the instructions in the file at the Effects section)
  3. In the language file: create an entry for the effect's tag. Warning: the entry in the language file must have the same name than the config object

The new effect will be automatically added to the list of effects in the configuration panel.

CSS

2 CSS files are necessary:

  • css/style.css : general properties of the slideshow
  • css/themes/default.css : theme

Themes

Several themes are delivered with AccesSlide in the folder css/themes.

<!-- Theme stylesheet -->
<link rel="stylesheet" href="css/themes/default.css" type="text/css" media="all" />

Post-processor

The CSS is generated by the post-processor myth.

Source files are in the css/sources folder and in the css/sources/themes folder for themes

Compiled files are in the css/themes folder for themes; and in the css/ folder for the general layout.

All the files are also available in a non-minified version to let you modify or create your own CSS without using a post-processor.

Automation with Grunt

A minimal grunt config is also available for CSS compilation. 4 modules are configured in Gruntfile.js:

  • grunt-myth : to compile in CSS
  • grunt-contrib-cssmin: to minify CSS
  • grunt-combine-media-queries: to combine media queries
  • grunt-contrib-watch

Customizing the interface

Icons (toolbar, configuration panel) are generated thanks to fontawesome.

Fallback en cas de non chargement de la police est permis grâce au script a font garde de Filament Group. Ce sont les images du dossier img qui prennent le relais en cas de non chargement.

Le css d'AccesSlide, n'embarque pas la bibliothèque fontawesome entièremment. Pour modifier une icône, référez vous à la documentation fontawesome et modifiez le fichier css d'AccesSlide avec le code voulu.


.icon-setting-sound:before{
	content: "\f028";
}

Adaptation accessibilité

Ces paramètres sont configurables via le panneau configuration. Les paramètres sont persistants (utilisation de cookies ou de localStorage si possible). Un bouton defaut permet de revenir à la configuration par défaut.

  • N° de diapo Vocalise les numéros des diapos
  • Textes masqués Emet un Bip lorsqu'un texte masqué est affiché
  • Diapo Emet un Bip à l'affichage d'une diapo
  • Diapo début Emet un Bip lors de l'affichage de la première diapo
  • Diapo fin Emet un Bip lors de l'affichage de la dernière diapo
  • Titre Vocalise le titre de la diapo courante
  • Titre fenêtre Met à jour le titre de la fenêtre lors de l'affichage de la diapo courante
  • Bouton suivant Donne le focus sur le bouton "suivant" lors du chargement du diaporama
  • Click Supprimer le clic pour aller à la diapo suivante.

Sounds library

Other parameters

  • Table of contents : Choisir le comportement du sommaire (modale ou non-modale). Dans le cas d'un sommaire non-modale, slides are resized.
  • Outline view: Linearizes the slideshow

Configure these parameters in the configuration panel.

Print

A print.css sheet permet une mise en page lors de l'impression via le navigateur (Ctrl + p)

L'impression embarque certains des styles du thème choisi (propriété all du css du thème)

Pour qu'un élément, ou une section entière n'apparaisse pas lors de l'impression, ajoutez une classe noprint

<section class="slide noprint">
 [contenu]
</section>

JavaScript

You can use your own scripts in the html page or in the slide.js file.

Outline view

The outline view vous permet d'afficher le diaporama sous la forme d'un contenu linéarisé, ce qui permet un travail plus rapide pour la vérification et la préparation des contenus

The outline view s'active via le configuration panel, des repères et le numéro des diapos indiquent le contenu de chaque diapo.

The outline view mode keeps the CSS styles of the slides but not the effects.

Tip!

Use two browsers, each in a different mode, to write and play with your slideshow.

Localization

All elements of the interface can be localized using a language file (lang folder).

To use a language file, edit the file path in the head of the page, for example <script type="text/javascript" src="lang/lang_fr.js"></script> for the French file.

Create a language file

  1. Open the language file with a text editor
  2. Edit the buttons labels label, image alternatives alt, options values value of the effects list, buttons or windows titles title, help messages help.
  3. Save your language file using the filename lang_[language code].js.
